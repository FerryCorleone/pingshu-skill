# 戏剧化与风格指南

## 风格目标

使用“现代高密度评书”：用评书的时机、停顿、转折和说书人态度，讲清楚现代事件。除非素材本身需要，不要写博物馆式古风。

目标效果是：

```text
清楚的爆点钩子 + 紧凑因果链 + 一路让人想笑的场面升级
```

不是：

```text
老派开场 + 反复口头禅 + 模糊剧情概括
```

也不是：

```text
完整背景 + 正确观点 + 反复升华 + 像在给观众上课
```

娱乐化内容的第一目标是好玩好笑。观点可以有，但必须像茶杯底那点余味，不能端到台前让听众先喝一大口。

## 导演层优先

评书化改编先是讲故事，再是有腔调。写正文前，先完成一张导演卡：

```text
一句话故事：谁在什么局里，因为哪个选择，欠下或补上什么账。
听众入场：没看过原素材的人，前 30 秒必须知道哪些背景。
娱乐承诺：这段主要好玩在哪，是荒唐流程、吃瘪、误会、错位还是名场面复盘。
笑点引擎：主要靠哪几种机制让人乐，不能只靠观点或感动撑住。
技巧编排：哪些 beat 用第一视角内心戏，哪些 beat 用一点传统评书味，哪些 beat 保持白描。
主问题：听众为什么继续听，想等哪件热闹事落地。
人物/关系弧线：谁从高位到低位，谁从信任到受伤，谁从犯错到补救。
冲突升级：触发、误读、代价、低点、反转、修复。
结尾落点：最后要给一个笑点、回扣、荒唐余韵或轻轻一酸；不要强行教育。
```

如果这张卡写不清楚，不要进入评书正文。否则技巧越多，听众越容易迷路。

## 技巧编排

把技巧当乐器，不当调料罐。每篇脚本先做 `technique_arrangement`：

```text
第一视角内心戏：最多 1-3 处，放在人物眼神、手、停顿、被打脸、误读规则、坏点子生成或临场装没事的瞬间。
传统评书味：最多 1-2 处，通常在开头定调、高潮前压一下、结尾轻轻收口。
留白段落：交代规则、推进事实、自救赶路、重要结果时，尽量直白，不要每句都耍。
```

好编排像一段音乐：开头有个调门，中段有一两个花活，高潮处有停顿，结尾有回扣。坏编排是每段都想证明自己会说书。

## 第一视角内心戏

这种写法适合把严肃人物、临场反应或荒唐选择讲活。它不是心理鉴定，也不是随便塞一句脑内弹幕，而是说书人把人物当场“怎么把这个念头想通”的过程喜剧化翻译出来。

适合使用：

- 人物看到关键物件，手已经伸过去，但嘴还没说；
- 人物发现事情不对，现场表情或动作明显卡住；
- 强势人物临场吃瘪、装镇定、想圆回来；
- 规则和常识冲突，人物需要一个荒唐但顺嘴的自我解释；
- 人物准备动歪心思，需要给自己找一个听上去“也不是不行”的理由。

写法骨架：

```text
可见动作 -> 当场压力 -> 角色一寻思/一琢磨 -> 第一视角短内心 -> 坏点子或吐槽落点 -> 立刻回到外部动作
```

示例：

```text
孙红雷手指头搭上箱扣，一寻思：节目规矩说能抢，弟弟又去洗手间了，箱子还搁我手边。我这时候不拿，待会儿别人拿了，我还落一好人牌，有啥用？先拎走，回头见面再说，哥跟你闹着玩呢。
```

坏例子：

```text
孙红雷一寻思：我来肯德基是吃套餐的，怎么旁边还配了个小金库？
```

问题是它只是在玩“肯德基 + 金库”的错位，没把人物为什么动歪心思、为什么拎箱走讲出来。

注意：

- 内心戏要短，像脑子里蹦出来的，不像独白演讲；
- 用 `一寻思`、`一琢磨`、`那意思像是`、`仿佛在说` 这类缓冲；
- 不写 `他心里就是`、`真实目的就是`；
- 每段内心戏必须能回答：他此刻想要什么？怕什么？用什么理由说服自己？下一步做了什么？
- 真人热点里更要克制，优先写“像是”，不要写成事实；
- 一篇中短脚本 1-2 处通常够了。

## 传统定场和小贯口

传统味不是古风滤镜，而是节奏、口条和台口。可以少量使用：

```text
闲言少叙，书归正传。
金条不大，压手；箱子不响，压心；肯德基灯一亮，桌上这点事儿可就藏不住了。
有话则长，无话则短，咱单表这只箱子。
```

适合使用：

- 开头 1-2 句定调，让听众知道“这是说书”；
- 转入关键场面前压住节奏；
- 结尾回扣醒木、箱子、门口、车钥匙等物件。

不适合使用：

- 每段开头都来一句；
- 用大段古典词遮住现代剧情；
- 写成对传统评书的拙劣模仿；
- 为了“有中国味”牺牲信息密度。

## 回目式标题

现代短篇也可以先起一个“这一回”的标题，帮节目定调。这里学的是章回体喜剧标题的结构，不是照搬任何现成剧集标题。

好标题通常有四个特点：

- 两句并列，中间用逗号或分号；
- 前半句写人物、物件或动作；
- 后半句写后果、误会、反转或另一个人的处境；
- 读起来文气一点，但观众能大概知道这回讲什么。

可用公式：

```text
人物/物件 + 动作，另一人物/局面 + 后果/反转
```

原创示例：

```text
艺兴暂托金条箱，红雷暗起撤退心
巴队一句难报姓名，日网三更怒开战
小赵怨写周报稿，龙虾错送全员群
```

开头可以这样入书：

```text
闲言少叙，书归正传。列位，今儿咱讲一回《小赵怨写周报稿，龙虾错送全员群》。
```

注意：标题是钩子，不是剧透。它可以点出物件和冲突，但不要一上来把核心反转、高潮动作和结局全讲完。

## 开场契约

开场要同时完成三件事：

1. 给一个具体反常画面，让人想听；
2. 用最少信息交代背景、人物关系和规则；
3. 许诺一个故事问题，而不是只摆气氛。

坏开场：

```text
列位，肯德基这个地方，一般管饱。这一回，管成长。
```

问题是：没看过原片的人不知道谁成长、为什么在肯德基、箱子从哪来。

好开场骨架：

```text
有一回《极限挑战》玩金条大战，规则很简单：箱子里是金条，谁守住谁占便宜。张艺兴刚跟孙红雷结盟，把箱子托给这位大哥，自己去了一趟洗手间。出来一看，箱子没了，车没了，大哥也没了。
```

这不是要求开场变成新闻导语，而是让听众先有地图，再看说书人怎么抖包袱。

## 名场面保真

名场面不是普通剧情概括。观众点名一个片段，通常是奔着一句话、一个眼神、一个物件、一组站位或一个反应来的。改编时要先问：

1. 这段最被转述的短台词是什么？
2. 观众脑子里最清楚的画面是什么？
3. 哪个动作或物件改变了权力关系？
4. 谁的反应让这段成立？
5. 哪个点不能被中文意译抹掉？

外语短名句如果就是梗本身，要让它在正文里原样出现。比如 `Say my name` 这类标题级台词，评书可以在前后用中文抬高、拆台、停顿、解释反差，但不能把它整句翻没了。正确做法是：短原句响一下，然后说书人接一句“好家伙，荒漠里突然上英语听力，题目就一个：谁才是这摊买卖的大哥。”

注意：保留短关键台词不等于复刻整段对白。长对白、完整场景调度和连续台词仍然要转述、评论和再创作。

## 场景复述优先

评书化不是把剧情梗概说得有腔调。已有剧集、动漫、综艺和采访片段，第一任务是把原场景讲出来。

坏写法像梗概：

```text
沃尔特提出反向方案，德克兰一开始拒绝，最后承认海森堡身份。
```

听众不知道谁先说话、谁占上风、为什么拒绝、承认之前发生了哪几拍，也就没法沉浸。

好写法按场景走：

```text
荒漠里车一停，德克兰那边人已经候着。人多，枪也在手边。麦克和杰西原本是来收摊的，老白却往前一站，先把这摊买卖改了题：材料不卖了，路子借我用，你们卖我的蓝货。
```

写作顺序：

1. 先建画面：地点、人物、站位、物件。
2. 再给动作：谁先走、谁先问、谁停顿、谁皱眉。
3. 再转对白：短名句可以原样保留，长对白只讲功能。
4. 再写反应：对方怎么接不住、怎么反问、怎么沉默。
5. 最后点变化：这一拍之后，权力关系或观众预期变了什么。

每个段落至少承担一个微镜头，不要只承担一个抽象结论。说书技巧、内心戏和包袱必须贴在微镜头上：手、眼神、车门、箱子、桌面、枪、沉默、反问、停顿。离开这些细节，技巧就会变成浮在上面的口头花活。

版权作品的边界是：不复刻长段台词，不逐字重建完整连续对白；但要忠实复述场景推进、对白功能和关键短句。观众要听得出来“这就是那场戏”，而不是“有人看完之后写的剧情简介”。

## 故事段落顺序

短篇名场面推荐用这个顺序：

| beat | 作用 | 常见问题 |
| --- | --- | --- |
| cold_hook | 用最反常的画面抓住人 | 只有俏皮话，没有事件 |
| context | 交代节目/事件规则、人物关系、目标 | 默认听众已经看过原片 |
| trigger | 哪个动作启动了后果 | 只说“后来出事了” |
| complication | 规则和情感开始冲突 | 把冲突写成空泛“很热闹” |
| low_point | 主角最难堪或最受伤的一刻 | 直接跳到结果，缺少代价 |
| turn | 人物做出选择或关系变化 | 只抖包袱，不推进 |
| resolution | 事情怎么收住 | 只有金句，没有事实结果 |
| aftertaste | 最后一个包袱、回扣或轻微余味 | 强行上价值，把娱乐内容讲成小作文 |

可以合并 beat，但不要省掉 `context`、`trigger`、`low_point` 和 `resolution`。

## 现代北方表演目标

当前产品方向更接近“一个北方说书人在现场讲一个戏剧性片段”，不是古典文本朗读。

目标是：

- 一个原创说书人声音；
- 现代、直白但有画面的中文；
- 紧凑动作和清晰因果；
- 稍粗粝、有调侃感、市井一点的旁白态度；
- 少量北方口吻；
- 利落停顿和短句爆发；
- 来自日常生活的比喻。
- 轻松、好玩、能转述给朋友的娱乐性。

目标不是：

- 多人广播剧；
- 泛化老年 AI 旁白；
- 古风套话；
- 满篇“列位看官”的戏服感；
- 用方言拼写本身当笑话；
- 铺垫很长但事实很少。
- 三五句就升华一次；
- 把综艺、动漫、热搜都讲成“人生道理”。

### 节奏画像

速度是场景结构，不是装饰标签。

| beat 类型 | 典型用途 | 写法 |
| --- | --- | --- |
| hold | 揭示、威胁、包袱前 | 短句，停顿，一个具体画面 |
| burst | 追逐、打斗、网友来回、快速矛盾 | 3-7 个带动词和物件的短句 |
| aside | 说书人拆台或降维 | 降低压力，用日常比较 |
| release | 包袱、反转、身份跌落 | 一句短促释放，然后停一下 |

做 TTS 计划时要交替使用这些 beat。一排 `medium` 段落说明方向还没写完。

### 事件级节奏

段落级 pace 不足以支撑成品听感。公开音频要把重要 beat 拆成 `say` 和 `pause` 事件。

- 多数 `say` 事件只承载一个画面、一个动作或一个包袱。
- 揭示前、反转后、旁白拆台前插入真实 `pause`。
- 350-550 ms 像逗号换气，650-900 ms 像压住眼神，1000-1200 ms 用于包袱或情绪落点。
- 语气词只有真正要被听见时才写出来：`欸。`、`嚯。`、`好家伙。`、克制的 `我操。`
- 不要把表演说明藏在正文里。`(停顿)`、`(吸气)`、`(意味深长)` 是渲染指令，不是台词。

## 场景转换流程

每个场景先做五步：

1. 找身份关系：谁高位，谁低位，谁暴露，谁惊讶，谁装没事。
2. 找冲突引擎：复仇、面子、误会、挑战、考验、诱惑、反转或荒唐流程。
3. 找感官细节：手、眼神、物件、距离、声音、停顿、围观反应。
4. 选择评书动作：抬高、喜剧降维、悬念延迟、说书人旁白、声音点或关子。
5. 写成 TTS 能干净渲染的短段。

## 微镜头动作展开

动漫、剧集、体育、综艺冲突要把视觉动作转成“微镜头”：

1. 谁注意到了什么；
2. 哪个物件或身体部位动了；
3. 它落到哪里，或者差点落到哪里；
4. 对手误读了什么；
5. 隐藏机关怎么露出来；
6. 身份关系如何变化；
7. 说书人如何抬高或拆台。

这能把“A 打了 B”这种平句变得可听，而不是靠空洞评书词硬撑。

## 段落密度规则

每段至少包含下面之一：

- 新的已核实事实；
- 因果链；
- 具体引语或主张的转述；
- 视觉或社会关系细节；
- 反转；
- 依赖事实成立的包袱。

只负责“听起来像评书”的段落，要删掉或重写。

只负责“显得有意义”的段落，也要删掉或重写。意义如果不能带来新的笑点、反转、动作或情绪释放，就先让位给场面。

## 观点降噪

娱乐化评书不是不能有观点，而是观点要短、轻、靠后。

优先写：

```text
他这一拎，节目组都省旁白了：大哥不是跑了，大哥是把肯德基吃成了地下车库撤离演习。
```

少写：

```text
这件事告诉我们，游戏可以赢一手，但信任欠下就要慢慢偿还。
```

降噪规则：

- 先写场面，再写判断；
- 判断一句够了，不要每段都结算一次；
- 少用 `人性`、`原则`、`成长`、`人生`、`意义`、`照明白`；
- 结尾优先回扣一个具体物件或动作，例如箱子、薯条、车钥匙、地铁扶手；
- 让听众自己感到一点东西，不要替听众把道理说满。

## 现代评书控制

- 开头最多用一次 `列位` 这类标记，结尾附近可以少量回扣。
- 大多数句子要直白、快、具体。
- 老派味用表演方向体现：停顿、速度、重音、醒木点、说书人挑眉。
- 方言是调料，不是主菜。
- 优先写“这事儿坏就坏在...”，不要套古典大词。

## 平淡动作展开

源素材只说“A 对 B 做了 X”时，用这个模式：

```text
1. 冻住现场。
2. 把动作抬高到值得听。
3. 延迟真正后果。
4. 加一个感官点。
5. 把受影响者的心理外化成表情或动作。
6. 让说书人评价。
7. 钩住下一步。
```

示例骨架：

```text
原始事实：A 打了 B 一巴掌。B 瞪回去。
改编 beat：
列位，这一巴掌在纸上写着轻巧，真到了当场，可就不是一只手的事。
[pause]
只见 A 肩头不动，腕子一翻，半空里像递来一封加急文书。
[sfx_after: waking_block]
B 先没说话，眼神往上一抬，那意思分明是：好小子，你拿我这张脸试鼓点来了？
```

## 笑点机制

一个 beat 最好只用一个清楚机制：

- **反差**：高调铺垫，低俗/日常落点。
- **吃瘪**：强势人物丢面子。
- **错位**：古典语气遇到现代物件，少量使用。
- **拆台**：说书人戳破过度严肃。
- **递进**：三层描述，第三层转弯。
- **误会**：观众知道角色没看懂。
- **正确反而可疑**：不该答对的人答得太顺。
- **生活化降维**：把超能力或大戏比成日常麻烦。

### 包袱开发流程

不要先写俏皮话。先把事实、预期和压力摆直，再找笑点。

```text
事实锚点 -> 普通预期 -> 角色压力 -> 可见动作 -> 生活化错位 -> 包袱落点 -> 后果
```

写作时至少问一遍：

1. 这个 beat 的已确认事实是什么？
2. 正常人会期待接下来发生什么？
3. 哪个身份、面子、规则或心理压力最可笑？
4. 有没有一个手、眼神、物件、距离或停顿能承载这个笑点？
5. 包袱能不能落在最后一个词、最后一个画面或停顿之后？
6. 抖完以后，剧情往哪继续走？

### 内心戏

可以让说书人替人物做“表演性心理翻译”，但只能从可见行为、身份关系和当场压力推出来。

可用句式：

```text
那意思像是...
你要搁现场看，大概就是这么个味儿...
仿佛在说...
```

真人热点和争议事件里，不要把内心戏写成真实动机。`他心里就是...`、`他早就打算...` 这类判断需要来源支持。

### 三段递进

从相声和脱口秀借用“三段式”：

```text
第一下建立正常预期。
第二下强化这个预期。
第三下突然转到荒唐但具体的生活细节。
```

第三下必须帮助理解当前剧情，不能为了凑笑话脱离事实。

### Callback

前文已经响过的词、物件或判断，后文可以轻轻回扣一次。回扣必须换语境，最好让它承担新的后果。

```text
前文：礼貌先下班了。
后文：等结果出来，礼貌又上班了，只不过这回网友开始加班。
```

原样重复不算 callback，只算水词。

## 风格护栏

- 不滥用“列位看官”；短篇里一两次足够。
- 不要每句都抖机灵。评书需要直线叙事，包袱才落得住。
- 不用晦涩方言影响理解。
- 不为了显得说书人聪明，就把原人物写蠢，除非来源确实如此。
- 现代互联网故事不要默认套 `三英战吕布`、`武松打虎` 这类模板。
- 不要用“网线两头很热闹”这种话遮住缺失细节。

## 热点事件 beat 模板

```text
1. 冷开场：最具体、最反常的爆点。
2. 触发点：谁说了/做了什么。
3. 解读：为什么 A 方觉得被冒犯。
4. 反击：为什么 B 方回嘴。
5. 升级：最好笑或最锋利的一来一回。
6. 结果：现实比赛/事件怎么落地。
7. 后续：最终变成什么梗或反讽。
```
