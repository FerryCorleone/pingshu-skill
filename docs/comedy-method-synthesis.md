# 包袱方法综合研究：评书 × 相声 × 脱口秀

## 目的

这份文档把三类相邻能力融到 `pingshu-skill` 的写作流程里：

- 评书：单人叙事、关子、动作描写、口头节奏和说书人判断；
- 相声：包袱组织、捧逗关系、说学逗唱、迟急顿挫；
- 脱口秀：setup/punch、misdirection、rule of three、callback、point of view。

目标不是把三种艺术混成杂烩，而是让 Agent 写现代评书时更会“造可听的戏”：事实清楚，动作有画面，人物有内心戏，包袱有铺垫和落点。

## 参考来源

- [北京评书 - 中国非物质文化遗产网](https://www.ihchina.cn/project_details/13685/)：一人坐讲，借手势、表情、口技交代情节和刻画人物，并形成“豪、紧、动、热”等特色。
- [相声 - 中国非物质文化遗产网](https://www.ihchina.cn/project_details/13669.html)：天津相声以说为主、以讽刺见长，风格火爆热烈，富于幽默感。
- [非遗中国：相声 - 新浪文化](https://culture.sina.cn/2019-03-19/detail-ihsxncvh3740693.d.html?vt=4)：相声吸收口技、说书等艺术之长，常用垫话、瓢把儿、正活儿、底等结构，用包袱使人发笑。
- [相声科普 - 北京旅游网](https://www.visitbeijing.com.cn/article/47QoCvH5WYY)：包袱来自语言、事件、情节的不协调，抖包袱需要作品理解、语气、手势、眼神和迟急顿挫。
- [Chris Head: Misdirection](https://www.chrishead.com/post/2018/02/02/lesson-8-misdirection)：误导是先建立预期，再在落点上推翻预期。
- [Chris Head: Rule Of Three, Callbacks & Anachronisms](https://www.chrishead.com/post/2018/04/27/lesson-17-comic-analogies)：三段式可理解为建立、强化、惊讶；callback 是把前面的材料换语境带回来。
- [Backstage: How to Use Callbacks in Your Comedy](https://www.backstage.com/magazine/article/callback-comedy-explained-76712/)：callback 能让作品更连贯，但要给听众时间，并且回扣时要扩展语境。
- [The Second City: Intro to Comedy Writing](https://www.secondcity.com/classes/virtual/online-classes/intro-comedy-writing-online-vir)：喜剧写作常训练观点、game、讽刺、可识别的人类行为和社会潜意识。

## 三种能力怎么分工

| 能力 | 负责什么 | 在本 Skill 里的落点 |
| --- | --- | --- |
| 评书 | 把故事讲成连续场面 | `story_pack` 的场景、动作、关子、拨口 |
| 相声 | 把不协调组织成包袱 | `pingshu_script` 的问答、递进、抖响 |
| 脱口秀 | 把观点压成明确笑点 | `comic_or_dramatic_handles` 和 rewrite loop |

现代评书脚本最容易失败的地方，是只学评书表面词，不学笑点和戏剧压力。正确顺序应该是：

```text
事实 -> 戏剧关系 -> 人物压力 -> 可见动作 -> 生活化错位 -> 包袱落点 -> 继续叙事
```

## Agent 包袱开发流程

每个重要 beat 写正文前，先做 7 步。

### 1. 找事实锚点

只从已确认素材里拿动作、原话、结果、画面和关系。没有事实锚点，就不能硬写包袱。

```text
错：两边吵得不可开交。
对：记者问谁最有威胁，球员一时说不出日本球员名字。
```

### 2. 找社会情绪

把事实翻译成普通人懂的情绪账：

- 被忽视；
- 被误会；
- 丢面子；
- 本来装稳，结果露怯；
- 想证明自己；
- 明明心虚还要嘴硬。

### 3. 选一个错位镜头

不要同时堆十种笑点。每个 beat 只选一个主镜头：

- 身份错位：高手像临时工；
- 规则错位：竞技像办手续；
- 情绪错位：大场面里有小心眼；
- 语言错位：宏大词落到家常话；
- 行动错位：角色以为自己很帅，观众看见他很狼狈。

### 4. 先写直的 setup

setup 先把事实和预期说清楚，不要提前把笑点抖出去。观众得先知道“正常应该怎样”，后面的反差才有劲。

```text
直 setup：巴西队员被问到日本队谁最有威胁。
预期：他说出一个名字，表示尊重。
可抖的点：他想了半天，现场空气先替他尴尬。
```

### 5. 用三段递进加压

三段式不是必须凑三句，而是让预期越来越清楚。

```text
第一下：他听懂了问题。
第二下：他确实认真想了。
第三下：想完以后，名字没出来，礼貌先下班了。
```

### 6. 抖在最后一个词或最后一个画面

包袱的笑点尽量放在句末、段末或停顿之后。不要在一句话中间提前解释。

### 7. 立刻回到剧情

笑点是推进工具，不是停车场。抖完马上交代后果：谁被冒犯，谁开始回嘴，事件怎么升级。

## 内心戏写法

内心戏最像用户说的“那个博主真正好笑的点”。它不是胡编人物心理，而是从可见行为里提炼一个临时的“表演性翻译”。

### 可用格式

```text
外部动作：他手伸到一半停住。
说书翻译：那意思像是脑子里开会了：拿吧，显得馋；不拿吧，白来。
回到事实：就这么一停，对面已经看出来他慌了。
```

### 真人边界

真人热点里，内心戏必须带表演性缓冲：

- `那意思像是...`
- `仿佛在说...`
- `你要搁现场看，大概就是这么个味儿...`

不要写成：

- `他心里就是看不起对方。`
- `他真实目的就是炒作。`
- `他早就设计好了。`

## 单人捧逗

评书是单人表演，但可以让说书人自己承担“逗哏”和“捧哏”。

```text
逗：您说这事儿坏在哪？
捧：坏就坏在，他不是没准备。
翻：他准备得挺充分。
抖：就是准备了半天，准备的是沉默。
```

这对 TTS 很重要：问句后要留 `pause`，回答时压低一点，抖包袱时再放出来。

## Callback 写法

callback 只能回扣已经有过效果的东西，而且必须换语境。

```text
前文包袱：礼貌先下班了。
后文回扣：等巴西赢完，礼貌又上班了，只不过这回轮到网友加班。
```

不要把一句话原样重复三遍。重复必须递进：

```text
第一次：建立词。
第二次：换场景。
第三次：变成结果或讽刺。
```

## 写作质量检查

一个现代评书 beat 写完后，用下面问题检查：

- 这个笑点是否必须依赖当前素材？换个故事还能用，就重写。
- 有没有先建立事实和预期，再抖包袱？
- 有没有至少一个可见动作、物件、眼神、距离或身体细节？
- 内心戏是不是从可见行为推出来的，而不是编造动机？
- 抽象类比是否帮助理解剧情，而不是盖过剧情？
- 包袱之后是否继续推进因果？
- 是否为 TTS 留了可执行停顿，而不是全靠模型自己猜？

## Agent 重写循环

给每个关键 beat 生成 5 个候选包袱：

1. 一个身份账本版；
2. 一个日常规则错位版；
3. 一个内心戏版；
4. 一个三段递进版；
5. 一个 callback 或后文伏笔版。

然后删掉：

- 不依赖素材细节的；
- 只靠粗口或方言的；
- 把真人动机写死的；
- 影响剧情清晰度的；
- TTS 念出来会别扭的。

留下最能同时满足“讲清楚”和“好笑”的版本。宁可一个 beat 只留一个真包袱，也不要每句都硬抖。
