# pingshu-storyteller 开源说明报告

## 一句话

`pingshu-storyteller` 是一个把“有来源的故事”变成“现代中文评书音频”的 Agent Skill。它面向普通用户，也面向想研究 Agent 创作工作流的开发者。

你可以把它理解成一个小型创作导演：它不只写稿，还会先找素材、整理场景、设计笑点、规划 TTS 节奏，并在音频生成后处理交付。

## 它解决什么问题

现在很多 TTS 模型已经能发出不错的声音，但真正难的地方不在“念出来”，而在：

- Agent 能不能先把故事找准；
- 能不能把名场面里的关键动作、台词、物件和反应留下来；
- 能不能把故事写成一场有节奏、有包袱、有画面的单人表演；
- 能不能让 TTS 按停顿、速度、重音去演，而不是一口气念完；
- 能不能把最终音频交付给用户，甚至放到用户平时听音频的软件里。

这个项目把这些步骤拆成可复用的结构化流程。

## 项目特色

### 1. 不是提示词，而是一条工作流

普通提示词往往会直接生成一段“像评书的文字”，结果容易空、老派、车轱辘话多。

这个 Skill 强制先产出：

- `story_pack.json`：素材、来源、人物、场景、冲突、名场面细节；
- `pingshu_script.json`：标题、导演方案、评书正文；
- `performance_plan.json`：TTS 表演计划；
- `delivery_plan.json`：音频交付和可选上传方案。

### 2. 先完整讲故事，再使用评书技巧

项目里有明确规则：不能为了追求评书、相声或脱口秀技巧，丢掉最基本的叙事能力。

一个好结果应该让没看过原片的人也能听懂：谁在场，谁想要什么，谁误判了，局面怎么变了，最后为什么好笑或过瘾。

### 3. 面向现代内容

它不只适合传统章回小说，也适合：

- 综艺名场面；
- 动漫和剧集片段；
- 采访冲突；
- 热点事件；
- 用户自己的故事；
- 适合娱乐化讲述的评论和解读。

### 4. 有 TTS 设备判断

如果用户想本地部署 TTS，Skill 会先检测设备，给出建议：

- 高质量默认档：VoxCPM2；
- 低配兜底档：Qwen3-TTS 0.6B；
- 配置太低：建议云端 API。

这样普通用户不需要先研究一堆模型参数。

### 5. 音频交付更完整

生成音频不是终点。Skill 会在正式音频完成后询问是否上传到网易云音乐云盘。用户不需要上传时，就直接交付本地文件；用户需要上传时，Agent 再引导登录和配置。

## 小白怎么安装

### 第一步：准备工具

你至少需要：

- Git；
- Node.js 18 或更高版本；
- 一个支持本地 Skills 的 Agent，例如 Codex。

如果只想先看文档，不需要任何模型。

### 第二步：下载项目

```bash
git clone https://github.com/FerryCorleone/pingshu-storyteller.git
cd pingshu-storyteller
```

### 第三步：安装 Skill

Codex 用户可以运行：

```bash
mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
ln -sfn "$(pwd)/pingshu-storyteller" "${CODEX_HOME:-$HOME/.codex}/skills/pingshu-storyteller"
```

然后重启或刷新 Codex。

### 第四步：验证

```bash
node pingshu-storyteller/scripts/validate_skill_outputs.mjs examples
```

如果看到校验通过，就说明示例产物结构正常。

## 怎么使用

你可以直接对 Agent 说：

```text
使用 pingshu-storyteller，把《绝命毒师》Say my name 这个名场面做成 3 分钟以内的现代评书音频。
```

更推荐你加上边界：

```text
使用 pingshu-storyteller。请先找来源，不要编剧情；保留名场面的关键短台词；风格要好玩，不要说教；TTS 先用本地 VoxCPM2。
```

## Agent 会经历什么流程

```text
用户目标
  -> 查来源 / 读素材 / ASR
  -> story_pack.json
  -> story_design
  -> pingshu_script.json
  -> performance_plan.json
  -> TTS 设备或 API 检查
  -> 音频渲染
  -> ASR 回检
  -> 本地交付或网易云上传
```

## 需要注意什么

### 版权

这个项目适合做评论、解读、戏仿、转述和个人收听场景，但不意味着你可以随便公开发布完整版权内容。公开发布前，请确认原作品、片段、平台和二创规则。

### 声音

不要用它克隆在世艺人、主播、普通人或明星的声音，除非你有明确授权。项目默认推荐原创说书人声音方向，而不是模仿某一个具体人。

### API key

不要把 API key、private key、cookie 或 token 写进产物文件、上传 manifest、日志或聊天记录。Skill 的脚本会尽量避免保存密钥，但你仍然要保护自己的凭据。

### 云盘上传

网易云音乐云盘上传只建议用于用户自己的私有收听。公开电台、播客、音乐作品发布，需要额外确认权利和平台规则。
