# 评书道具音效

## 结论

音频产品里只默认内置醒木音效。扇子、手绢、桌案等传统道具更多是现场表演的身段和视觉提示，不适合在纯音频里硬做“扇子声”“衣料声”。醒木要像标点，不要像鼓点：能定场、留扣、收束，但不能替代叙事和包袱。

## 传统依据

中国非遗数字博物馆的[北京评书](https://www.ihchina.cn/project_details/13686.html)条目提到，评书通常一人坐在桌后讲述，常用折扇、醒木等道具，并借助手势、表情和口技增强节奏与气氛。[四川评书](https://www.ihchina.cn/project_details/13755.html)条目也把醒木、扇子、手绢列为常见辅助道具。

行业经验文章常见的说法是：旧书场里醒木常用于开书、收书和关口；现代舞台更宜少用，常放在开场定场、开场韵白之后、转折留扣或结尾收束。随便敲会分散听众注意力，让人以为有新情节或新段落。

## 可用资产

当前 Skill 只内置一个本机生成的 48kHz mono WAV：

- `assets/sfx/waking_block.wav`：醒木，0.355 秒。源文件后部空白已裁掉，只保留敲击本体和短桌面余响。

资产来源和裁剪参数记录在 `assets/sfx/manifest.json`。公开分发前要按 Stable Audio 当前许可再确认可打包范围。

## 使用规则

默认位置：

- 开场短定场和回目标题之后，可用一次 `waking_block`。
- 重大转折前，如果前后有足够留白，可用一次 `waking_block`。
- 结尾如果是“下回分解”式留扣，或轻巧收束，可用一次 `waking_block`；短节目已经开头用过时，结尾通常不再用。

频率预算：

- 6 分钟以内：最多 2 次，默认 1 次。
- 6-15 分钟：最多 3 次。
- 15 分钟以上：约每 4-5 分钟最多 1 次。
- 两次醒木之间至少间隔 45 秒；不要连续两个 segment 都用。

不要使用的位置：

- 不要给每个包袱、每句台词、每个动作都敲。
- 不要在人物对白中间敲，除非 performance_plan 明确把对白前后拆成事件并留出真实静音。
- 不要用醒木掩盖文字无聊、剧情缺细节或节奏没设计的问题。
- 不要把“啪”“醒木一响”写进 TTS 正文，除非这是说书人要念出来的比喻句。真实音效应在后期插入。

## performance_plan 写法

在要插入音效的段落或事件上使用 `sfx_after`：

```json
{
  "id": "seg-001",
  "text": "闲言少叙，书归正传。列位，今儿咱讲一回《某某某》。",
  "pause_after_ms": 450,
  "sfx_after": ["waking_block"]
}
```

事件级计划也可以放在某个 `say` 事件后：

```json
{
  "type": "say",
  "text": "列位，您听这事儿邪不邪？",
  "sfx_after": ["waking_block"]
}
```

渲染器必须把音效当成后处理音频块插入，不送进 TTS 文本。默认混音目标是“能听见，但低于人声”，不要让醒木把说书人压没。

节奏处理也交给渲染器：醒木后默认留约 420ms，如果计划里停顿太短则拉到约 320ms 以上，如果太长则压到约 650ms 以内。事件级显式 `pause` 可覆盖这个默认节奏。
